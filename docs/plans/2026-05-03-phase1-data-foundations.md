# План: Phase 1 — данные для индексов и RAG (Shinjuku)

**Дата:** 2026-05-03  
**Тип:** Новая функция (фундамент данных)  
**Роли:** Architect → Implementer → Guardian → Tester  

## Цель

Подготовить SQLite маппинг почтовых индексов из `KEN_ALL_ROME.CSV` и Markdown-базу правил для района Синдзюку с учётом изменений апреля 2024 (令和6年4月).

## Объём работ

1. Модуль нормализации ромадзи муниципалитета → `district_id` и загрузка в `data/zip_codes.db`.
2. Скрипт извлечения текста из PDF (`000378895.pdf`) через PyMuPDF4LLM и курируемый `knowledge/shinjuku_rules.md`.
3. Фаза Qdrant/docker (`PHASE_1_DATA.md` §3) — не входит в данный спринт (агент DevOps).

## Риски

| Риск | Митигация |
|------|-----------|
| Неверная кодировка CSV | Чтение только как `shift_jis`; падение при ошибке декодирования с понятным сообщением |
| Дубликаты индексов в CSV | `INSERT OR REPLACE` — сохраняется последняя строка при конфликте PK |
| PDF отсутствует локально | Скрипт поддерживает URL загрузки официального файла |

## Миграции / Rollback

- БД генерируется скриптом: rollback = удалить `data/zip_codes.db` и перезапустить скрипт.
- Markdown и извлечённые файлы — под версионным контролем или перегенерация из PDF.

## Шаги реализации

1. Добавить пакет `garbage_data` (нормализация, без глобальных синглтонов).
2. Реализовать `scripts/process_zip_codes.py` (аргументы: путь CSV, путь БД).
3. Реализовать `scripts/extract_shinjuku_pdf.py` для необязательной перегенерации черновика из PDF.
4. Сформировать `knowledge/shinjuku_rules.md` с разделами Burnable / Non-burnable / Resources (Plastic) / Sodai и блоком про апрель 2024.

## Rollback

Удалить артефакты сборки (`data/zip_codes.db`, опционально `knowledge/generated/*`) и откатить коммит с кодом.
