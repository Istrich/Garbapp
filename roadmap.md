Этап 1: Фундамент и обработка данных (Data Engineering) — **выполнен**
Прежде чем писать код приложения, нужно подготовить данные, на которых будет учиться ИИ.

- **Индексы:** скрипт конвертации `KEN_ALL_ROME.CSV` → SQLite `data/zip_codes.db`, быстрый поиск района по индексу.
- **RAG-подготовка:** Markdown-база `knowledge/shinjuku_rules.md` (в т.ч. правило апреля 2024), опционально черновик из PDF через `scripts/extract_shinjuku_pdf.py`.
- **Векторизация:** Docker Compose с Qdrant, загрузка чанков и метаданных `district_id` через `scripts/init_vector_db.py`.
- **Интеграционная проверка цепочки:** `scripts/integration/test_full_chain.py` (индекс `1600022` → анализ фото сценария «пластиковое ведро»).

Этап 2: Разработка Backend ядра (AI Pipeline)
На этом этапе вы создаете «мозг», который принимает фото и выдает вердикт.

FastAPI Setup: Настройка базового сервера и эндпоинтов для приема почтового индекса и изображений.

Vision Chain: Реализация первого запроса к GPT-4o для получения структурного описания предмета (материал, размер, чистота).  

Logic Chain: Реализация связи между результатами Vision, почтовым индексом и правилами из RAG.  

Тестирование: Проверка точности классификации на наборе тестовых фото (например, то самое черное ведро).  

Этап 3: Кроссплатформенная мобильная разработка (Frontend)
Создание интерфейса, с которым будет взаимодействовать мигрант.

Логика локации: Экран ввода и сохранения почтового индекса.

Модуль камеры: Интеграция плагина камеры во Flutter с наложением рамки-подсказки для пользователя.

UX/UI: Создание понятных карточек результата: категория мусора, иконка, цвет (например, зеленый для ресурсов) и краткая инструкция.  

Этап 4: Интеграция и облачная инфраструктура (DevOps)
Соединение фронтенда с бэкендом и подготовка к нагрузке.

Docker: Упаковка бэкенда, векторной базы и базы данных в контейнеры для легкого деплоя.

API Security: Настройка защиты эндпоинтов и обработки ошибок (например, если ИИ не смог распознать предмет).

Cloud Deploy: Развертывание системы на сервере (AWS/Hetzner) и настройка HTTPS.

Этап 5: Расширение и локализация (Scaling)
Добавление новых районов и улучшение точности.

Масштабирование RAG: Добавление новых городов (Осака, Фукуока, Саппоро) путем простой загрузки их PDF в базу данных.  

Календарь событий: Интеграция расписания вывоза мусора, чтобы приложение напоминало: «Завтра день сбора пластика».  

Мультиязычность: Добавление японского и английского интерфейсов для охвата большего числа экспатов.
