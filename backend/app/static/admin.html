Админка

Один сценарий: указываете почтовый индекс — backend находит district_id в zip_codes.db, сохраняет PDF и индексирует чанки с подписью района; при проверке фото снова даёте тот же индекс — RAG подтягивает правила именно этого района.

Нужен для вкладок «Индексация PDF», «База индексов (KEN)» и «Промпты ИИ».

Если указан — PDF сохраняется в data/sources/<district из БД>/, ingest помечает чанки тем же district_id, что и в zip_codes.db. Только если индекс пустой.
Ответ API

Загрузите официальный файл KEN_ALL_ROME.CSV (как в папке проекта Index base/). Таблица zip_mapping в data/zip_codes.db будет полностью пересоздана из файла (старые строки удаляются). Кодировка оригинала Japan Post — обычно shift_jis.

Большой файл (сотни тысяч строк) может обрабатываться до нескольких минут — не закрывайте вкладку до ответа сервера. На Railway без volume для /workspace/data база пропадёт при редеплое — при необходимости смонтируйте volume на каталог данных.

Ответ API
Если указан — район берётся из zip_codes.db (приоритет). Формат: 0600031 или 060-0031. Используется только если поле индекса пустое — должен совпадать с district_id в Qdrant после ingest.
